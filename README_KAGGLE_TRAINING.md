# 🏗️ ArchVision AI - Навчання на Kaggle Датасетах

## Огляд

ArchVision AI тепер підтримує дообучення на архітектурних датасетах з Kaggle використовуючи математику з **ArchiLense фреймворку**. Нова архітектура включає:

- **VLM (Vision-Language Model)**: ViT-B/16 + CLIP text encoder
- **Контрастивні функції втрат**: InfoNCE, Triplet Loss, Center Loss, Focal Loss
- **Локалізацію ознак** для виявлення архітектурних елементів
- **Підтримку Kaggle датасетів** для автоматичного завантаження та навчання

## 📊 Підтримувані Датасети

### Основний: Architectural Styles Dataset
- **Джерело**: [Kaggle - dumitrux/architectural-styles-dataset](https://www.kaggle.com/datasets/dumitrux/architectural-styles-dataset)\n- **Стилів**: 25 архітектурних стилів\n- **Зображень**: ~10,000 зразків\n- **Якість**: Професійні фото архітектури\n\n### Додаткові датасети (опціонально):\n- `architectural-style-classification`\n- `historical-architecture`\n- `building-style-recognition`\n\n## 🚀 Швидкий Старт\n\n### 1. Налаштування Kaggle API\n\n```bash\n# Встановіть kaggle API\npip install kaggle\n\n# Створіть Kaggle API ключі на https://www.kaggle.com/settings\n# Скачайте kaggle.json та помістіть у ~/.kaggle/ (Linux/Mac) або %USERPROFILE%\\.kaggle\\ (Windows)\n```\n\n### 2. Налаштування змінних оточення\n\nСтворіть `.env` файл:\n\n```env\n# Kaggle API\nKAGGLE_USERNAME=your_username\nKAGGLE_KEY=your_api_key\n\n# Hugging Face (для базових моделей)\nHUGGINGFACE_TOKEN=your_hf_token\n\n# Google API (для збору додаткових даних)\nGOOGLE_API_KEY=your_google_api_key\n```\n\n### 3. Запуск навчання\n\n```bash\n# Автоматичне завантаження датасету та навчання\npython kaggle_trainer.py\n\n# Або з кастомними параметрами\npython -c \"from kaggle_trainer import KaggleTrainingPipeline; pipeline = KaggleTrainingPipeline('./data', batch_size=16, num_epochs=30); pipeline.train()\"\n```\n\n## ⚙️ Конфігурація\n\n### Параметри навчання (в `kaggle_trainer.py`):\n\n```python\nconfig = {\n    'data_path': './datasets/architectural_styles',    # Шлях до датасету\n    'model_save_path': './models/archvision_kaggle',  # Де зберігати модель\n    'batch_size': 16,                                  # Розмір батчу\n    'learning_rate': 1e-4,                            # Швидкість навчання\n    'num_epochs': 30,                                 # Кількість епох\n    'kaggle_dataset': 'dumitrux/architectural-styles-dataset'\n}\n```\n\n### Гіперпараметри VLM архітектури:\n\n```python\narchvision_vlm_config = {\n    'embedding_dim': 768,      # Розмірність embeddings\n    'temperature': 0.07,       # Температура для contrastive loss\n    'margin': 0.2,            # Margin для triplet loss  \n    'center_loss_weight': 0.005,  # Вага center loss\n    'num_classes': 83,        # Кількість архітектурних стилів\n}\n```\n\n## 🧮 Математичні Основи (ArchiLense)\n\n### Комбінована функція втрат:\n```\nL = α·L_contrast + β·L_ce + γ·L_center + δ·L_triplet + ε·L_focal\n```\n\nДе:\n- **α = 1.0** - Contrastive loss (InfoNCE)\n- **β = 1.0** - Cross-entropy loss\n- **γ = 0.005** - Center loss\n- **δ = 0.2** - Triplet loss\n- **ε = 0.5** - Focal loss\n\n### InfoNCE Contrastive Loss:\n```\nL_contrast = -1/N * Σᵢ log(exp(s(i,i)/τ) / Σⱼ exp(s(i,j)/τ))\n```\n\n### Triplet Loss (Hard Mining):\n```\nL_triplet = Σᵢ max(0, d(aᵢ,p) - d(aᵢ,n) + margin)\n```\n\n### Center Loss:\n```\nL_center = 1/2 * Σᵢ ||vᵢ - c_yᵢ||²\n```\n\n## 📈 Процес Навчання\n\n### Етапи:\n1. **Завантаження датасету** з Kaggle\n2. **Аугментація** зображень (rotation, flip, color jitter)\n3. **Контрастивне навчання** image-text pairs\n4. **Fine-tuning** на архітектурних стилях\n5. **Валідація** та збереження найкращої моделі\n\n### Аугментації (згідно ArchiLense):\n- Multi-scale training: 224→384→512\n- Random rotation: ±10°\n- Color jitter: brightness, contrast, saturation\n- Horizontal flip: 50%\n- RandomResizedCrop: scale=(0.8, 1.0)\n\n## 📊 Моніторинг Навчання\n\nСкрипт автоматично генерує:\n- **training_history.json** - історія втрат та точності\n- **training_plots.png** - графіки навчання\n- **best_model.pth** - найкраща модель\n- **best_metadata.json** - метаданні моделі\n\n## 🎯 Використання Навченої Моделі\n\nПісля навчання модель автоматично підвантажується в основний аналізатор:\n\n```python\n# У ml_engine.py модель шукається автоматично:\nkaggle_model_paths = [\n    \"./models/archvision_kaggle/best_model.pth\",\n    \"./models/archvision_kaggle/checkpoint_model.pth\", \n    \"./kaggle_models/best_model.pth\",\n    \"./pretrained/archvision_vlm.pth\"\n]\n```\n\n## 🔧 Налагодження\n\n### Часті Проблеми:\n\n**1.
